# Skills 目录安全判断指南

> 用于约束 GitHub 自动发现候选 Skill 时的安全与恶意风险判断。

## 核心原则

自动发现候选 Skill 时，需要同时做安全风险初筛。这个判断只用于提醒维护者，不代表仓库一定安全或一定恶意。

自动发现不能直接把候选写入 `data.js`。候选必须先进入 `candidates/skills_candidates.json`，由维护者人工确认后，再进入正式卡片生成流程。

## 候选字段

后续候选结果应增加：

- `risk_level`：可取 `low`、`medium`、`high`、`unknown`
- `risk_reasons`：记录触发风险判断的原因，方便人工复核

## 风险信号

高风险信号：

- README 诱导执行不明脚本
- 存在明显混淆代码
- 出现可疑下载链接
- 要求输入密钥、token 或敏感凭据
- 权限说明异常，超出功能所需范围

中风险信号：

- 仓库信息过少
- 无 README
- 无 License
- 维护记录很少
- 功能描述与代码结构明显不一致

低风险信号：

- 仓库结构清晰
- README 说明完整
- 维护记录正常
- 代码结构和功能描述基本一致

## 实现要求

- 风险较高的候选不应自动写入 `data.js`
- 候选列表必须保留风险原因，而不是只给一个分数
- 风险判断只作为维护者决策辅助，不能替代人工确认
- 页面端不承担安全扫描逻辑
